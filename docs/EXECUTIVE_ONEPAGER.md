# stack-mcp-server — one-pager для руководства

**HTML-версия (для показа и печати):** [EXECUTIVE_ONEPAGER.html](./EXECUTIVE_ONEPAGER.html) — встроенные SVG-схемы и пиктограммы, мало текста; откройте в браузере.

**Назначение:** один экран / одна страница, удерживающая внимание ~8 секунд и вызывающая желание углубиться.

---

## Каркас слайда (F-паттерн)

```
┌─────────────────────────────────────────────────────────────────┐
│  [ВИЗУАЛЬНЫЙ ЯКОРЬ — метафора / одна сильная фраза]     [логотип] │
│                                                                  │
│  ЗАГОЛОВОК-УДАР (до 6 слов)                                      │
│  Подзаголовок — один результат для бизнеса / мира                │
│                                                                  │
│  One-liner ценности (одна строка)                                  │
│                                                                  │
│  ┌ Было → Стало          ┐     [КРУПНАЯ ЦИФРА + доказательство]  │
│  ├ Было → Стало          ┤                                       │
│  └ Было → Стало          ┘     [CTA — нестандартный призыв]       │
│                                                                  │
│  Слоган 2–3 слова                                                │
└─────────────────────────────────────────────────────────────────┘
```

**Фон (для дизайнера):** глубокий графит → ночной синий (`#0B0F14` → `#121826`); один акцент — янтарь / тёплый белый (`#E8C07D` или `#F4F1EA`) только на якоре и цифре. Много воздуха, без визуального шума.

---

## 1. Визуальный якорь (0,5 с — захват)

**Образ (не иллюстрация — эмоция):**  
*«Один штурвал — весь зоопарк данных.»*

**Смысл:** единый мост между ИИ-агентом и OpenSearch, Kafka, PostgreSQL, Redis, Prometheus — всё подключено, каждый рычаг с предохранителем.

**Эффект:** контроль и порядок вместо хаоса интеграций.

---

## 2. Заголовок-удар (1–2 с)

**Заголовок:**  

### Один MCP. Все данные. Ноль хаоса.

**Подзаголовок:**  
Единый HTTP‑MCP с включаемыми модулями и жёсткими лимитами — агенты получают доступ к фактам, не к продакшену целиком.

**Эффект:** контраст «всё / ноль» + понятный результат (риск ↓, скорость решений ↑).

---

## 3. One-liner ценности (2–3 с)

**Формула:**  
Мы помогаем **лидерам данных и платформенным командам** достичь **скорости ИИ‑операций без новых «чёрных ящиков» и аварий** — один контур, одна политика, одна кнопка выключения.

**Эффект:** «Да, это про нас» + снятие страха перед автономными агентами.

---

## 4. Триплет «До / После» (3–4 с)

Вертикальный блок слева (под F-паттерном). Не технологии — **чувства**.


|       | Было                                      | Стало                                           |
| ----- | ----------------------------------------- | ----------------------------------------------- |
| **1** | Долго собирать картину вручную            | Одна сводка по данным из всех модулей           |
| **2** | Непонятно, кто и что вызывал у систем     | Журнал вызовов и доступ строго по политике      |
| **3** | Интеграции на месяцы                      | Включили модуль — работает                      |


**Эффект:** левая колонка удерживает взгляд; продаётся ощущение, не список фич.

---

## 5. Социальное доказательство + метрика (4–6 с)

**Яркая цифра (подставьте пилот):**  
**1** шлюз · **N** модулей · **1** конфиг  

*Альтернатива после пилота:* «−***% времени на разбор инцидента» / «+*** обращений к данным в смену без эскалации».

**Доказательство:**  
Единый репозиторий, воспроизводимые сценарии деплоя, документированные возможности (`docs/CAPABILITIES.md`).

**Почему цепляет:** «1» читается как простота; «N модулей» — масштаб без размножения сервисов; лимбика получает *«мы не усложняем — мы упрощаем»*.

---

## 6. CTA + слоган (7–8 с)

**Призыв (не «купить»):**  
**Вместо 15** минут метаний администратора — 1 минута AI-агента: собрал данные, дал аналитику.  
*или:* *Включите демо на наших логах — без продакшена.*

**Слоган (2–3 слова):**  

### Данные на поводке.

**Эффект:** интрига и безопасное любопытство → естественный следующий шаг (воркшоп, пилот).

## Сводка нейро-дизайна (для презентации дизайнеру)


| Правило          | Как выполнено                                                                                              |
| ---------------- | ---------------------------------------------------------------------------------------------------------- |
| F-паттерн        | Якорь и заголовок сверху слева → триплет вниз слева → цифра и CTA вторым фокусом справа или под заголовком |
| 70 / 20 / 10     | ~70% — образ + главная цифра; ~20% — заголовок; ~10% — подписи и триплет                                   |
| Эмоции > функции | Контроль, скорость, отсутствие стыда перед советом директоров — не «MCP и HTTP»                            |
| 8 секунд         | Цепочка: узнавание → облегчение → масштаб → желание спросить «как у нас это внедрить?»                     |
