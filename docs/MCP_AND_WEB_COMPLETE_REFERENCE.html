1. Архитектура: что считать «MCP-сервером»

В проекте один процесс протокола MCP под именем FastMCP sdocs-mcp (CLI sdocs-mcp): набор инструментов (tools) зависит от YAML modules.*.enabled. Отдельный процесс sdocs-mcp-ui — это не отдельный MCP-сервер, а веб-приложение (FastAPI) с дашбордом, JSON API и Prometheus /metrics; при SDOCS_MCP_EMBED_MCP=true оно монтирует тот же набор MCP tools по HTTP на путь /mcp.

Итого «логических MCP-серверов» по смыслу заказчика: один (sdocs-mcp), плюс опционально тот же экземпляр внутри UI на /mcp.

5. Встроенный MCP на {base}/mcp

При SDOCS_MCP_EMBED_MCP=true (значение по умолчанию в Docker-образе UI) на том же порту, что и UI, монтируется приложение Streamable HTTP MCP по пути /mcp или <SDOCS_MCP_UI_BASE_PATH>/mcp, если задан префикс UI (в коде: build_mcp(cfg, streamable_http_path="/") + app.mount(...)). Клиенты MCP подключаются к URL вида http(s)://<host>:<ui-port>/mcp или с префиксом.

Выключить встроенный MCP: SDOCS_MCP_EMBED_MCP=false и запускать отдельный процесс sdocs-mcp на порту SDOCS_MCP_PORT (по умолчанию 8765).

При SDOCS_MCP_UI_WORKERS>1 и встроенном MCP без SDOCS_MCP_STATELESS_HTTP=true сессии Streamable HTTP могут ломаться — см. предупреждение в логах UI.

6. TLS и mTLS (сервер MCP и UI)

По умолчанию и UI, и процесс sdocs-mcp слушают обычный HTTP без TLS.

Включить HTTPS + обязательный клиентский сертификат (mTLS) для uvicorn: задать одновременно все переменные (пути к существующим файлам):

  • SDOCS_MCP_MTLS_CERT_FILE — серверный сертификат (CRT)
  • SDOCS_MCP_MTLS_KEY_FILE — приватный ключ (KEY)
  • SDOCS_MCP_MTLS_ROOT_CA_FILE — CA для проверки клиентских сертификатов
  • опционально SDOCS_MCP_MTLS_KEY_PASSWORD — пароль к зашифрованному ключу

Реализация: resolve_mcp_mtls_uvicorn_kwargs() в mtls.py — при неполном наборе переменных или отсутствии файлов возвращается None и в лог пишется предупреждение; сервер остаётся на HTTP.

Выключить mTLS/TLS: снять или очистить любую из трёх обязательных переменных SDOCS_MCP_MTLS_* (или удалить файлы) и перезапустить процесс — снова будет plain HTTP.

Для отдельного процесса sdocs-mcp используется тот же механизм в server.py (uvicorn с ssl kwargs).

7. Клиентский mTLS к бэкендам (PostgreSQL, Redis, Kafka, Prometheus, OpenSearch)

Задаётся в YAML модуля: либо все три поля mtls_cert_file, mtls_key_file, mtls_root_ca_file (и опционально mtls_key_password), либо ни одного. Иначе клиент ходит к бэкенду как раньше (plain/TLS по URL/настройкам модуля). Для Redis при клиентском mTLS URL должен быть rediss://. Комментарии в config.example.yaml § строки 13–17.
